# Glossary: NVIDIA Isaac AI-Robot Brain

This glossary defines key terms used throughout the NVIDIA Isaac AI-Robot Brain module.

## A

**A\*** - A graph traversal and path search algorithm that uses heuristics to find the optimal path from a start node to a goal node.

**Action Server** - A ROS 2 component that accepts goals from clients, processes them, and returns results along with feedback during processing.

## B

**Balance Control** - Systems and algorithms that maintain the stability of a humanoid robot by managing its center of mass and support polygon.

**Behavior Tree** - A hierarchical structure used in robotics for organizing and executing complex behaviors in a modular way.

**Bipedal Locomotion** - The act of walking on two legs, characteristic of humanoid robots.

## C

**CUDA** - Compute Unified Device Architecture, NVIDIA's parallel computing platform and programming model.

**Costmap** - A 2D or 3D representation of the environment that assigns costs to areas based on obstacle proximity and other factors.

**Center of Mass (CoM)** - The point in a body where the total mass of the body is considered to be concentrated.

## D

**Deep Learning** - A subset of machine learning that uses neural networks with multiple layers to model complex patterns.

**DWB (Dynamic Window Approach)** - A local planning algorithm that samples possible robot motions and selects the best one based on criteria like obstacle avoidance and goal reaching.

## F

**Footstep Planning** - The process of determining where and when to place feet during bipedal locomotion to maintain stability.

**Forward Kinematics** - The computation of the position and orientation of a robot's end effector based on joint angles.

## G

**Global Planner** - A path planning algorithm that computes a path from start to goal considering the global map of the environment.

**GPU (Graphics Processing Unit)** - A specialized electronic circuit designed to rapidly manipulate and alter memory to accelerate the creation of images in a frame buffer intended for output to a display device.

## H

**Hardware Acceleration** - The use of specialized hardware to perform specific tasks more efficiently than software running on a general-purpose CPU.

**Humanoid Robot** - A robot with human-like characteristics, particularly in terms of appearance and behavior.

## I

**Isaac ROS** - A collection of hardware-accelerated perception and navigation packages that bridge NVIDIA's GPU computing platform with the Robot Operating System.

**Isaac Sim** - A GPU-accelerated simulation application for robotics that provides high-fidelity physics and photorealistic rendering capabilities.

## L

**Local Planner** - A path planning algorithm that generates short-term trajectories considering immediate obstacles and robot dynamics.

**LiDAR** - Light Detection and Ranging, a remote sensing method that uses light in the form of a pulsed laser to measure distances.

## M

**Motion Planning** - The computational problem of finding a sequence of valid configurations to move an object from a starting state to a goal state.

## N

**Nav2** - The Navigation2 stack, a state-of-the-art navigation system for ROS 2.

**Neural Network** - A series of algorithms that endeavors to recognize underlying relationships in a set of data through a process that mimics the way the human brain operates.

## P

**Path Planning** - The computational process of determining a path from a starting position to a goal position.

**Perception Pipeline** - A sequence of processing steps that transform raw sensor data into meaningful information about the environment.

**Photorealistic Rendering** - The creation of images that are visually indistinguishable from photographs of real-world objects and environments.

## S

**SLAM (Simultaneous Localization and Mapping)** - The computational problem of constructing or updating a map of an unknown environment while simultaneously keeping track of an agent's location within it.

**Sensor Fusion** - The process of combining data from multiple sensors to achieve improved accuracy and reliability.

**Sim-to-Real Transfer** - The ability to transfer skills or models trained in simulation to real-world robots.

**Synthetic Data Generation** - The creation of artificial data that mimics real-world data for training machine learning models.

## T

**Tensor Cores** - Specialized processing units in NVIDIA GPUs designed to accelerate deep learning and scientific computing operations.

**TensorRT** - NVIDIA's inference optimizer and runtime for deep learning applications.

**Trajectory** - A time-parameterized path that specifies how a robot should move through space over time.

## V

**VSLAM (Visual Simultaneous Localization and Mapping)** - SLAM that uses visual sensors as the primary source of information.

**Voxel** - A volume element, representing a value on a regular grid in three-dimensional space.

## Z

**ZMP (Zero Moment Point)** - A point on the ground where the net moment of the ground reaction force is zero, used as a stability criterion for humanoid robots.
